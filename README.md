# Git Pull Request Demo

This repository was created as part of a GitHub practice exercise to learn the Git workflow.

## Objective

The purpose of this repository is to practice:

- Creating a GitHub repository.
- Creating multiple branches.
- Making changes in different branches.
- Opening a Pull Request.
- Merging branches after review.

## Academy

This task was completed as part of the NextFlows AI Academy training.

Academy Website:
https://nextflows.ai/

## Branches

- **main** – The primary branch.
- **feature-1** – Practice branch for Git operations.
- **feature-2** – Practice branch used to create and merge a Pull Request.

## Workflow Completed

1. Created a new GitHub repository.
2. Cloned the repository to the local machine.
3. Created two branches (`feature-1` and `feature-2`).
4. Updated the README file in both branches.
5. Pushed both branches to GitHub.
6. Opened a Pull Request.
7. Reviewed and merged the Pull Request successfully.

## How to Run This Project

1. Clone the repository:

```bash
git clone https://github.com/Tala-Saabneh/git-pr-demo.git
```

2. Navigate to the project folder:

```bash
cd git-pr-demo
```

3. Create a new branch (optional):

```bash
git checkout -b feature-name
```

4. Make your changes.

5. Stage and commit the changes:

```bash
git add .
git commit -m "Describe your changes"
```

6. Push the branch to GitHub:

```bash
git push -u origin feature-name
```

7. Open a Pull Request on GitHub and merge it into the **main** branch after review.
